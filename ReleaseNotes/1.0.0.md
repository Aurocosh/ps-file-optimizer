# 1.0.0

First public release of **PS-FileOptimizer** — a PowerShell module and CLI that runs the same external plugin optimization chains as [FileOptimizer](https://sourceforge.net/projects/nikkhokkho/files/FileOptimizer/), with scriptable cmdlets, JSON configuration, and undoable backups.

Requires Windows PowerShell 5.1+ or PowerShell 7+.

## What you can do

- Optimize files in batch from the shell or scripts (`Optimize-FoFile` / `Scripts/Optimize-File.ps1`)
- Preview planned plugin steps without changing files (`-WhatIf`)
- Install the pinned plugin tool bundle automatically (`Install-FoPlugins` / `Scripts/Install-Plugins.ps1`)
- Keep a history of successful optimizations and roll them back (`Get-FoHistory`, `Undo-FoOptimization`)
- Inspect pipelines and execution plans (`Get-FoPipeline`, `Get-FoExecutionPlan`)
- Configure defaults globally or per run (`Initialize-FoConfig`, `Get-FoConfig`, `-ConfigPath`)

## Format coverage

**39 pipeline groups** cover images, documents, archives, audio/video containers, and more — mapped from extension via `Data/ExtensionMap.psd1`, with content-based detection where needed.

Examples include PNG, JPEG, GIF, WebP, TIFF, PDF, ZIP/GZIP/7z, MP3/FLAC/OGG, MP4/MKV, EXE/DLL, HTML/CSS/JS, and others. Unrecognized image-like types can optionally use the catch-all **MISC** pipeline (ImageMagick); disable it with `MiscDisable = true` if you do not want that behavior.

## Plugin tools

Optimization uses external plugin binaries (the same class of tools FileOptimizer ships), not a plugin API.

- Download a verified zip from [ps-file-optimizer-aux](https://github.com/Aurocosh/ps-file-optimizer-aux) into `Plugins64\` or `Plugins32\`
- Modes: **FullPortable** (full install), **Missing** (fill gaps), **Remove**
- Search modes: portable folder and/or PATH (`PortableFirst`, `PathFirst`, `PortableOnly`, `PathOnly`)
- Steps that need a missing tool are skipped when possible; with `SkipMissingTools = true`, a file is skipped entirely if any required tool is missing

32-bit (`Plugins32`) omits a few x64-only tools (`minify`, `optivorbis`, `tinydng-cli`); related pipelines still run with fewer steps.

Plugin binaries have their own licenses — see [`THIRD_PARTY_NOTICES.md`](../THIRD_PARTY_NOTICES.md) and the aux repo [`PLUGIN-CREDITS.md`](https://github.com/Aurocosh/ps-file-optimizer-aux/blob/master/PLUGIN-CREDITS.md).

## Output modes and safety

By default, optimization keeps a backup of the original (`OutputMode = TempMove` under `%TEMP%\FileOptimizer\backups`). Other modes:

| Mode | Behavior |
|------|----------|
| `TempMove` | Optimized file at the original path; original moved to a temp backup (default, reversible) |
| `BackupSuffix` | Original renamed beside the optimized file (reversible) |
| `BackupMove` | Original moved under a backup tree (reversible) |
| `OptimizedSuffix` | Original kept; smaller copy written with a suffix (reversible by deleting the copy) |
| `Replace` | Overwrite in place (not reversible) |

A step’s output is kept only when it is meaningfully smaller than the input (FileOptimizer-style keep-if-smaller). Use `-ContinueOnError` to finish a multi-file batch after individual failures.

## Configuration

Settings merge in order: module defaults → `%USERPROFILE%\.config\FileOptimizer\config.json` → optional `-ConfigPath` → cmdlet parameters.

`Initialize-FoConfig` writes a starter config from the module template. Tunables include optimization level (0–9), plugin path/search mode, history, timeouts, and per-format options (for example lossy PNG/JPEG/WebP flags, PDF profile, metadata copy for several formats).

## Cmdlets and CLI

| Cmdlet | Purpose |
|--------|---------|
| `Optimize-FoFile` | Optimize one or more files |
| `Install-FoPlugins` | Download/install/remove plugin bundles |
| `Get-FoConfig` / `Initialize-FoConfig` | Read or create configuration |
| `Get-FoHistory` / `Undo-FoOptimization` | History and rollback |
| `Get-FoPipeline` / `Get-FoExecutionPlan` | Inspect chains and plans |
| `Invoke-FoPluginChain` | Run the chain for a single file |
| `Resolve-FoPluginExecutable` | Locate a plugin tool |
| `Install-FoDssim` | Optional DSSIM install for **image tests only** (not used by optimization) |

Matching scripts under `Scripts\` wrap the same operations for day-to-day CLI use (`Optimize-File.ps1`, `Install-Plugins.ps1`, `Show-History.ps1`, `Undo-Optimization.ps1`, and others).
