# 1.0.2

Bug fix and plugin-bundle format release.

## Plugin bundle 1.1.0

- New archive layout keeps `Plugins64/` / `Plugins32/` and adds root-level `fo-plugin-bundle.json` (bundle version + per-file SHA256).
- Aux release also publishes sidecar `*.manifest.json` copies of that file.
- `Install-FoPlugins` verifies manifest hashes and copies `fo-plugin-bundle.json` into the install folder.
- Optimize runs require installed bundle ≥ `1.1.0`. Older/unversioned plugin trees fail with a hard error unless you pass `-AcknowledgeOutdatedPlugins` (persists `AcknowledgedPluginBundleMinimum` until the module raises the minimum again; acknowledged runs warn and continue).
- Default plugin location remains `{ModuleRoot}\Plugins64` / `Plugins32`.

## Fixes

- Stale config `PluginPath` falls back to the loaded module's plugin folder.
- Clearer messaging when no tools ran because plugins are missing (instead of "already optimal").
- `Initialize-FoConfig` no longer bakes a versioned absolute Gallery plugin path.
- Unit coverage for stale `PluginPath` fallback during settings merge.
