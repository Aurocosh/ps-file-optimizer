# 1.0.1

Bugfix release.

## Fixes

- **DefluffPipe hang** — `Invoke-FoDefluffPipe` no longer deadlocks on larger inputs. It previously wrote the entire file to stdin before draining stdout; once the OS pipe buffer filled (~64KB), PNG/GZIP/ZIP chains could stick on the defluff step indefinitely. Stdin, stdout, and stderr now copy concurrently.
- **Gzip recompress** — the compress half of `Invoke-FoGzipRecompress` uses the same concurrent pipe pattern to avoid the same class of hang.
- **Handler timeout cleanup** — timed-out handler processes are killed as a process tree (so `cmd.exe /c` stubs cannot leave child tools running), and timeout paths no longer leak stray pipeline output.

## Tests

- Added a regression test that streams >64KB through a pipe echo stub to ensure DefluffPipe does not deadlock.
